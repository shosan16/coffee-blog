`src/app/recipes/[id]/page.tsx` を構成する各コンポーネントについて、「改善」または「置き換え」を前提に、実装レベルでの修正タスクを整理してください。

## 参考にしてほしいモック

- `mock/recipe-detail-mock.html`

## 対象コンポーネント

- Hero セクション
- バリスタセクション
- 抽出条件セクション
- 使用器具セクション
- 抽出手順セクション

## やってほしいこと

- 各コンポーネントごとに、改善・置き換えの目的を一文で整理する
- 実装作業として分解可能なタスクを洗い出す
- 各タスクはチェックボックス形式の todo とする
- UI / 文言 / データ構造 / コンポーネント分割など、観点が分かるように整理する

## 出力形式

- コンポーネント単位で見出しを分ける
- todo は GitHub Markdown 形式（`- [ ]`）で記述する
- `docs/client/improve-detail-page/todo.md` に出力

## 制約

- 抽象的な方針や感想ではなく、実際にコード修正ができるレベルの具体性で書く
- 不要な説明文は書かず、目的と todo に集中する
